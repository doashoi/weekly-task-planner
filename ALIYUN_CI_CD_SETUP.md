# 阿里云代码托管平台配置说明

## CI/CD 配置文件说明

本项目提供了多种适用于阿里云代码托管平台的 CI/CD 配置文件：

### 1. .gitlab-ci.yml (推荐)
- 兼容 GitLab CI 标准的配置文件
- 适用于大多数支持 GitLab CI 格式的平台
- 使用 SSH 进行服务器部署
- 包含构建和部署两个阶段

### 2. pipeline.yaml
- 阿里云原生流水线配置文件
- 使用阿里云 DevOps 特定语法
- 更好地集成阿里云服务

### 3. aliyun-pipeline.yml
- 专为阿里云优化的流水线配置
- 包含阿里云特定的环境变量和插件

## 部署流程说明

当前部署流程包括以下步骤：

1. **代码拉取**：从代码仓库拉取最新代码
2. **依赖安装**：在构建环境中安装项目依赖
3. **远程部署**：通过 SSH 连接到目标服务器执行部署脚本
4. **服务重启**：更新代码后重启应用程序

## 必需的环境变量

在阿里云代码托管平台中需要配置以下变量：

- `HOST`：目标服务器 IP 地址
- `PORT`：SSH 连接端口
- `USERNAME`：服务器登录用户名
- `SSH_PRIVATE_KEY`：SSH 私钥，用于连接服务器

## 部署目标服务器要求

目标服务器必须满足以下要求：

- Node.js 环境（版本 14+）
- PM2 进程管理器
- Git 工具
- 对应的 SSH 公钥已添加到服务器

## 故障排除

如果部署失败，请检查：

1. 环境变量配置是否正确
2. SSH 私钥是否有足够权限
3. 目标服务器是否在线且防火墙配置正确
4. 目标服务器上的目录是否存在且有适当权限
